## 1. コンセプト

現状の町は「電池を使う → 種類を選ぶ → 自動配置 → 棟数が増える」が中心で、町らしさ・情緒が薄い。

目標:

> 歩くたびに、静かな町に灯りと暮らしが増えていく

派手なゲーム演出より、次を重視する。

- 時間帯で変化する町
- 建物を建てた瞬間の手応え
- 人が暮らしている気配
- 発展の節目を記憶に残す演出

**やらないこと（本計画の範囲外）**:

- 手動配置 UI
- 隣接ボーナスなどの戦略ルール
- 建物アップグレード
- Flame 等のゲームエンジン導入
- オンライン天気 API

## 6. 技術・品質の共通ルール

詳細は [town-atmosphere-spec.md §0](town-atmosphere-spec.md#0-共通制約必読) を参照。要約:

- Flame 等は導入しない。Flutter 標準アニメーションのみ
- 見た目だけの状態は永続化しない（例外: 演出済み段階・町名など明示したもののみ）
- ゲームバランス数値（コスト・容量・係数）は変更しない
- 新規ウィジェットは `lib/widgets/town/` に置く
- 時刻・乱数は注入可能にしてテストする
- 各 Phase 完了時: `flutter analyze` / `flutter test` 全パス
