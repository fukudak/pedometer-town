# 万歩計タウン 技術スタック

**バージョン**: 0.1  
**日付**: 2026-06-15

---

## 確定スタック

| 項目           | 技術             | 理由 |
|----------------|------------------|------|
| フレームワーク | Flutter + Dart   | 健康データアクセスが最も安定。最初からネイティブアプリとして動作 |
| 健康データ     | `health` パッケージ | Health Connect / HealthKit両対応が容易 |
| 状態管理       | Provider         | MVPでは十分。シンプルで学習コストが低い |
| ローカル保存   | SharedPreferences | 軽量で十分な永続化 |
| UI             | Material Design  | 最初はミニマルに。後でカスタム可能 |

---

## 将来検討事項

- 状態管理の移行（Riverpod / Bloc）
- より高度なUI（CustomPainter / Flame）
- バックエンド導入の是非

---

## 選定理由

- Reactの資産を気にしない前提で選定
- モバイルの歩数取得を最初から本気でやるため、Flutterが最も適していると判断
- 開発速度とメンテナンス性のバランスを重視
