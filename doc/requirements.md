# 万歩計タウン 要件定義書

**バージョン**: 1.0  
**日付**: 2026-06-27

---

## 1. 目的

実際の歩行データをゲームに変換し、体重・速度を考慮した「移動エネルギー」を蓄電池に蓄え、町を発展させる放置・育成ゲームを作る。

---

## 2. コアコンセプト

- 現実の歩行 → ゲーム内資源
  - 歩数 + 移動スピード + 体重 → 「移動エネルギー」を算出
  - エネルギーは蓄電池に蓄積
- 蓄電池 → 町発展
  - 蓄電池が満タンになるとストックされ、町画面で消費して建物が自動建設される
  - 建物の種類・効果により蓄電池容量や発電効率が向上する
- 放置ゲーム寄り
  - 歩くだけで自然にエネルギーが溜まる
  - プレイヤーは「満タン蓄電池をいつ町に使うか」を楽しむ

---

## 3. 主要要件

### 3.1 入力・計算
- 体重・歩行速度・発電変換係数を設定可能
- スマホから歩数を取得（iOS: HealthKit / Android: ハードウェアセンサー）
- GPS による歩行速度の計測（設定画面）
- 移動エネルギーを計算し蓄電池に蓄積

### 3.2 町システム
- 蓄電池満タン分を消費して建物を自動建設（住宅・発電所・公園）
- 建物には効果がある（容量アップ、効率アップ、人口）
- 建物数に応じた発展段階表示（地平線ビュー）
- 最終段階到達後はロケット発射演出

### 3.3 進行
- 日ごとの歩数・発電量記録（履歴画面）
- 蓄電池満タン・ロケット発射・実績解除のイベント記録
- 町レベル（建物数）・文明スコア

### 3.4 非機能要件
- 完全にオフラインで動作
- データは端末内に保存（SharedPreferences）
- クリア条件なし（無限発展型）

---

## 4. スコープ

**実装済み**
- 体重・速度・発電係数の設定
- iOS / Android からの歩数取得
- エネルギー計算・蓄積・満タンストック
- 建物の自動建設（3種類）と効果反映
- 発展段階・実績・履歴画面
- GPS 歩行速度計測

**将来検討**
- 手動での建物配置 UI
- 難易度プリセット
- オンライン機能

---

## 5. 成功基準

- アプリをインストールして歩数を取得できる
- 歩行でエネルギーが蓄積し、満タン蓄電池で町が発展する
- データが端末に保存され、再起動後も残る
