# 万歩計タウン 実装計画

**バージョン**: 0.3  
**日付**: 2026-06-15  
**ベース**: 要件定義書 v0.1

---

## フェーズ分け

### Phase 1: 基盤 + スマホ歩数取得（最優先）

**目的**: Flutterプロジェクトを作成し、スマホから実際の歩数を取得できる状態を作る

**タスク**:
1. Flutterプロジェクト初期化
2. `health` パッケージ導入
3. 体重設定画面（SharedPreferences保存）
4. 歩数取得 + 権限リクエストの実装
5. エネルギー計算ロジック実装（物理値 + 難易度係数）
6. 蓄電池表示

**完了条件**:
- スマホの歩数データが取得できる
- エネルギーが計算・蓄積される
- データが永続化される

---

### Phase 2: 町の基礎（MVP）

**目的**: エネルギーを使って建物を建てられる状態にする

**タスク**:
1. 建物データ定義（3〜5種類）
2. 町画面（リスト表示）
3. 建物建設機能
4. 建物の効果反映

**完了条件**:
- エネルギーを消費して建物を建てられる

---

### Phase 3: 基本的な進行感

**目的**: 毎日遊べる感覚を出す

**タスク**:
1. 日付ごとの歩数記録
2. 連続歩行ボーナス
3. 町レベル表示

---

### Phase 4: 将来拡張の土台

**タスク**:
1. 難易度切り替えUI
2. 高度なHealthデータの活用検討

---

## 未決定事項

- 難易度の種類と係数値
- 1日のエネルギー上限の扱い
